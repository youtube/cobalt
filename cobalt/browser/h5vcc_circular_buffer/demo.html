H5vccCircularBuffer Demo & Manual Test

What is this?

This page demonstrates the usage of the H5vccCircularBuffer interface for reliable logging.

The goal of this circular buffer is to survive application crashes. Data written to it is persisted in a memory-mapped file.

Manual Test Instructions

  1. Load this page. It will check the buffer.
  2. If empty, it will automatically write a unique string (e.g., demo_data_[timestamp]).
  3. To test crash resilience: Open a terminal and hard kill Cobalt using kill -9 [PID].
  4. Restart Cobalt and load this page again.
  5. You should see the data written in step 2 displayed in the "Startup Check" section!
